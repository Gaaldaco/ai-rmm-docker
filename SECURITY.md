# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| latest  | Yes       |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

If you discover a security vulnerability in ai-rmm-docker, please report it privately:

1. **Email**: aldacogene@gmail.com
2. **Subject line**: `[SECURITY] ai-rmm-docker — brief description`
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will acknowledge your report within **48 hours** and aim to release a fix within **7 days** for critical issues.

## Scope

This policy covers:
- The API server (`api/`)
- The web dashboard (`web/`)
- The agent binary (`agent/`)
- Docker Compose configuration and startup scripts
- Any credentials, tokens, or secrets handling

## Responsible Disclosure

We ask that you:
- Allow us reasonable time to fix the issue before public disclosure
- Do not access, modify, or delete data belonging to others
- Do not exploit the vulnerability beyond what is necessary to confirm it
